# JMI-OPENATOM 成员主页与个人博客需求设计文档

## 1. 文档信息

| 项目名称 | 成员主页与个人博客 | 版本号 | V1.0.0 |
| --- | --- | --- | --- |
| 所属系统 | OpenAtom System 官网 | 文档状态 | 待评审 |
| 目标终端 | Web PC、平板、手机 | 主要用户 | 社团成员 |
| 前端技术栈 | Vue 3、TypeScript、Element Plus | 后端技术栈 | Spring Boot、MyBatis-Plus、MySQL、Redis |

## 2. 项目背景

当前 `/profile` 页面同时承担个人资料、申请进度、密码修改和第三方账号绑定等功能，内容以“账号管理”为主，无法作为社团成员对外展示自己的个人主页。

本功能将“公开展示”和“私密账号设置”彻底拆分：

- 成员拥有一个可被社团内其他成员访问的个人主页，用于自我介绍、展示博客、项目、技能和社团经历。
- 新增成员主页广场，通过用户自定义的横幅长条卡片展示所有有效成员。
- 成员可从系统提供的组件和主题中编排自己的主页，不允许执行任意脚本或不受控 CSS。
- OAuth 绑定、手机号、学号、登录记录等账号信息只允许本人查看，不进入任何公开主页接口。

## 3. 产品目标

- 为每位有效社团成员建立统一、可分享的成员主页。
- 将现有博客文章与作者主页关联，形成“关于我 + 个人博客”的内容空间。
- 允许成员在安全范围内定制主页布局、内容、主题和横幅名片。
- 让社团成员更容易发现其他成员及其技术方向、项目和文章。
- 从后端响应层隔离公开数据与私密账号数据，避免敏感字段误泄露。
- 保持现有官网的深浅色主题、响应式体验和设计语言。

## 4. 核心原则

### 4.1 公开内容必须由用户主动发布

系统不得默认公开手机号、邮箱、学号、班级或第三方账号绑定信息。公开主页只展示用户在主页编辑器中明确启用的模块和字段。

### 4.2 私密信息不能只靠前端隐藏

公开主页接口必须使用独立的公开 VO，不得直接返回 `User`、`ResponseCurrentUserVO` 或账号设置对象后再由前端隐藏字段。

### 4.3 绑定身份和公开社交链接相互独立

- `google_sub`、QQ OpenID、GitHub/Gitee OAuth subject 属于登录凭据，只能本人访问。
- 主页中的 GitHub、Gitee、个人网站等属于用户主动填写的公开链接。
- 绑定 GitHub 或 Gitee 后不得自动在公开主页展示对应账号。

### 4.4 使用受控组件，不开放任意代码

主页支持预置组件、主题、排序、宽度和可见性配置，但不允许用户提交任意 HTML、JavaScript、iframe 或全局 CSS。

## 5. 用户角色与权限

| 角色 | 成员广场 | 查看成员主页 | 编辑自己的主页 | 查看私密设置 | 管理违规主页 |
| --- | --- | --- | --- | --- | --- |
| 未登录访客 | 不允许 | 不允许 | 不允许 | 不允许 | 不允许 |
| 已登录但非有效成员 | 可按运营策略关闭 | 默认不允许 | 不允许 | 仅自己的账号设置 | 不允许 |
| 有效社团成员 | 允许 | 允许 | 允许 | 仅本人 | 不允许 |
| 管理员 | 允许 | 允许 | 仅自己的主页 | 仅自己的账号设置 | 允许隐藏、恢复或审计 |

“有效社团成员”建议定义为存在有效社团成员关系，且成员状态为正式、试用或系统允许展示的状态。离社、封禁或注销成员默认不出现在成员广场。

## 8. 成员主页广场

### 8.1 页面目标

通过统一的成员长条卡片，让成员快速了解其他人的身份、方向和内容，并进入其个人主页。

### 8.2 横幅长条卡片

每位有效成员必须拥有一张卡片。用户未完成自定义时，系统使用默认背景和安全的默认内容。

卡片推荐比例：

- PC：宽高比约 `4:1`，最小高度 180px。
- 平板：宽高比约 `3:1`。
- 手机：改为 `16:9` 或紧凑纵向卡片，避免文字拥挤。

卡片内容：

| 内容 | 是否必填 | 规则 |
| --- | --- | --- |
| 背景图 | 否 | 用户上传；没有时使用系统默认背景 |
| 头像 | 是 | 使用公开头像或默认头像 |
| 展示名称 | 是 | 用户公开昵称优先，否则使用安全默认名称 |
| 一句话介绍 | 否 | 建议 12～40 个中文字符 |
| 部门 / 职位 | 否 | 由用户选择是否公开 |
| 技术标签 | 否 | 最多展示 4 个 |
| 博客数量 | 否 | 仅统计已发布文章 |
| 访问按钮 | 是 | 进入 `/members/:slug` |

卡片结构示意：

```text
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [头像]  展示名称                         [访问主页 →]        │
│          一句话介绍                                          │
│          前端 · Java · 开源实践      12 篇文章 · 技术部成员   │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 8.3 背景图片规则

- 支持 JPG、PNG、WebP，推荐 WebP。
- 建议尺寸不低于 1600×400px，宽高比推荐 4:1。
- 单张最大 8MB，服务端必须校验 MIME、文件签名和尺寸。
- 保存原图的同时生成适合卡片展示的压缩版本。
- 卡片上必须叠加可读性遮罩，文字对比度达到 WCAG AA。
- 用户可调整图片焦点位置 `focusX`、`focusY`，避免关键内容被裁切。
- 管理员可以隐藏违规背景，但不能读取或修改用户私密账号信息。

### 8.4 搜索与筛选

- 按展示名称、技术标签、部门搜索。
- 支持按部门、年级或技术方向筛选；仅使用被允许公开的数据。
- 默认按最近更新或系统推荐排序。
- 成员数量较多时使用服务端分页，不一次性加载全部数据。

## 9. 成员个人主页

### 9.1 页面定位

个人主页是成员在社团内部的“关于我 + 个人博客”空间。页面重点是个人介绍、技术内容和社团贡献，不承担账号设置功能。

### 9.2 固定页面元素

- 主页横幅与头像。
- 展示名称和一句话介绍。
- 返回成员广场入口。
- 页面所有者可见的“编辑主页”按钮。
- 最新更新时间。

### 9.3 可选主页组件

| 组件 key | 组件名称 | 主要内容 | 推荐默认可见性 |
| --- | --- | --- | --- |
| `about` | 关于我 | 个人简介、兴趣与方向 | 成员可见 |
| `skills` | 技能与方向 | 技术标签、熟练度或方向 | 成员可见 |
| `blog_latest` | 最新文章 | 已发布博客文章列表 | 成员可见 |
| `blog_featured` | 精选文章 | 用户手动选择的文章 | 成员可见 |
| `projects` | 项目作品 | 项目名称、简介、链接和图片 | 成员可见 |
| `timeline` | 成长时间线 | 社团经历、学习节点 | 成员可见 |
| `club_experience` | 社团经历 | 部门、职位、参与内容 | 用户选择 |
| `awards` | 荣誉与奖项 | 已有奖项数据或自定义内容 | 用户选择 |
| `activity_stats` | 活动足迹 | 可公开的活动参与统计 | 用户选择 |
| `social_links` | 公开链接 | GitHub、Gitee、个人网站等 | 用户主动填写 |
| `markdown` | 自定义内容 | 经过安全过滤的 Markdown | 成员可见 |

### 9.4 博客整合

- 复用现有博客文章和审核流程，不另外创建第二套文章系统。
- 个人主页只展示 `published` 状态的文章。
- “最新文章”按发布时间倒序展示。
- “精选文章”由用户从自己的已发布文章中选择，建议最多 6 篇。
- 点击文章进入现有 `/blog/:id` 页面。
- 博客详情页的作者区域增加“访问作者主页”入口。

## 10. 主页编辑器

### 10.1 编辑模式

PC 端推荐三栏结构：

```text
┌──────────────┬──────────────────────────────┬──────────────────┐
│ 组件库       │ 页面实时预览                 │ 当前组件设置     │
│ 关于我       │ [横幅]                       │ 标题             │
│ 技能         │ [关于我]                     │ 宽度             │
│ 博客         │ [最新文章]                   │ 可见性           │
│ 项目         │ [技能]                       │ 背景 / 样式      │
└──────────────┴──────────────────────────────┴──────────────────┘
```

移动端不使用三栏拖拽编辑，改为“组件列表 + 上移/下移 + 编辑抽屉”。

### 10.2 编辑能力

- 从组件库添加组件。
- 启用、隐藏或删除可选组件。
- 拖拽排序；移动端提供上移和下移按钮。
- 设置组件宽度：整行、半行；手机端统一整行。
- 修改组件标题和允许编辑的内容。
- 选择系统主题、强调色和卡片密度。
- 上传主页横幅和成员卡片背景图。
- 实时预览桌面与手机效果。
- 保存草稿、预览、发布。
- 页面发布采用版本号，更新冲突时提示用户刷新，避免覆盖其他标签页修改。

### 10.3 主题系统

首期建议提供 4 套受控主题：

1. `minimal`：黑白灰、内容优先。
2. `tech`：深色科技感、蓝紫弱强调。
3. `warm`：暖色背景、生活化表达。
4. `editorial`：大标题和杂志式文章布局。

主题只改变设计令牌，不允许用户提交任意 CSS。可配置项包括：

- 主题 key。
- 明暗模式偏好：跟随系统、浅色、深色。
- 系统提供的强调色。
- 组件圆角等级。
- 内容密度：舒适、标准、紧凑。

## 11. 可见性与隐私设计

### 11.1 可见性等级

| 等级 | 含义 |
| --- | --- |
| `private` | 仅本人可见 |
| `members` | 仅有效社团成员可见 |
| `unlisted` | 不出现在成员广场，但拥有地址且满足访问权限时可访问 |
| `hidden` | 管理员隐藏，不允许普通成员访问 |

首期默认使用 `members`，暂不建议开放互联网完全公开状态。

### 11.2 字段权限矩阵

| 数据 | 本人 | 其他有效成员 | 管理员业务接口 | 公开主页接口 |
| --- | --- | --- | --- | --- |
| 展示名称、公开头像 | 可见 | 用户启用后可见 | 可见 | 可返回 |
| 横幅、简介、技能、项目 | 可见 | 用户启用后可见 | 可审核 | 可返回 |
| 已发布博客 | 可见 | 可见 | 可审核 | 可返回 |
| 部门、职位、社团经历 | 可见 | 用户启用后可见 | 可见 | 按配置返回 |
| 手机号、邮箱、学号、班级 | 可见 | 不可见 | 按既有权限访问 | 永不返回 |
| 密码哈希、刷新令牌 | 不直接返回 | 不可见 | 不直接返回 | 永不返回 |
| QQ OpenID、Google sub | 仅显示“已绑定/未绑定” | 不可见 | 不返回原值 | 永不返回 |
| GitHub/Gitee OAuth subject | 仅显示绑定状态 | 不可见 | 不返回原值 | 永不返回 |
| 登录时间、登录日志、IP | 本人安全页按需显示 | 不可见 | 审计权限可见 | 永不返回 |

### 11.3 私密设置页

`/settings/account` 只允许当前登录用户访问，包含：

- 账号基础资料。
- 密码修改与退出登录。
- Google、GitHub、Gitee、QQ 绑定状态及绑定操作。
- 隐私设置。
- 会话管理和安全记录的后续扩展入口。

页面不得提供通过 URL 参数查看其他用户账号设置的能力。

## 12. 数据结构建议

### 12.1 `member_profile`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT | 主键 |
| `user_id` | INT UNIQUE | 用户 ID |
| `slug` | VARCHAR(64) UNIQUE | 主页短地址，只允许小写字母、数字和连字符 |
| `display_name` | VARCHAR(80) | 公开展示名称 |
| `headline` | VARCHAR(160) | 一句话介绍 |
| `bio` | TEXT | 个人简介 |

| `banner_url` | VARCHAR(500) | 个人主页横幅 |
| `card_background_url` | VARCHAR(500) | 成员广场长条卡片背景 |
| `card_focus_x` | DECIMAL(5,2) | 背景焦点 X，0～100 |
| `card_focus_y` | DECIMAL(5,2) | 背景焦点 Y，0～100 |
| `theme_key` | VARCHAR(32) | 受控主题 key |
| `color_mode` | VARCHAR(16) | system/light/dark |
| `visibility` | VARCHAR(16) | private/members/unlisted/hidden |

| `published_at` | DATETIME | 发布时间 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

### 12.2 `member_profile_module`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT | 主键 |
| `profile_id` | BIGINT | 主页 ID |
| `module_key` | VARCHAR(32) | 组件类型 |
| `title` | VARCHAR(80) | 自定义标题 |
| `sort_order` | INT | 排序 |
| `column_span` | TINYINT | 6 或 12，表示半行或整行 |
| `enabled` | BOOLEAN | 是否启用 |
| `visibility` | VARCHAR(16) | private/members |
| `config_json` | JSON | 组件受控配置 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

约束：`UNIQUE(profile_id, module_key)`。同类组件如需多实例，应增加 `instance_key`。

### 12.3 `member_profile_social_link`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT | 主键 |
| `profile_id` | BIGINT | 主页 ID |
| `platform` | VARCHAR(32) | github/gitee/website 等 |
| `label` | VARCHAR(60) | 展示文本 |
| `url` | VARCHAR(500) | 用户主动填写的公开链接 |
| `sort_order` | INT | 排序 |
| `enabled` | BOOLEAN | 是否展示 |

该表不得引用或复制 `user_external_identity.subject`。

### 12.4 博客关联

复用现有博客文章的作者用户 ID。查询个人主页文章时，以主页的 `user_id` 关联已发布文章，不新增重复关联表。精选文章可使用：

- `member_profile_featured_article(profile_id, article_id, sort_order)`；或
- 在 `blog_featured` 模块的 `config_json` 中保存文章 ID，并在服务端校验文章属于当前用户且状态为已发布。

首期推荐独立关联表，便于约束和查询。

## 13. API 设计建议

统一前缀沿用 `/api/v1`。

### 13.1 成员访问接口

| 方法 | 路径 | 权限 | 说明 |
| --- | --- | --- | --- |
| GET | `/members` | 有效成员 | 分页获取成员长条卡片 |
| GET | `/members/{slug}` | 有效成员 | 获取成员公开主页 |
| GET | `/members/{slug}/articles` | 有效成员 | 获取成员已发布文章 |
| GET | `/members/meta/filters` | 有效成员 | 获取可用部门、标签筛选项 |

### 13.2 本人编辑接口

| 方法 | 路径 | 权限 | 说明 |
| --- | --- | --- | --- |
| GET | `/me/profile` | 本人 | 获取主页草稿和全部本人配置 |
| PUT | `/me/profile` | 本人 | 更新基础展示信息和主题 |
| PUT | `/me/profile/layout` | 本人 | 保存组件排序和布局 |

## 15. 安全要求

- 所有成员页接口必须在服务端校验登录状态和有效成员身份。
- `/members/{slug}` 使用专用公开 VO，禁止序列化 `User` 实体。
- Markdown 必须过滤危险标签、事件属性、`javascript:` URL 和 iframe。
- 外部链接只允许 `https`，必要时允许受控 `http`；打开新页面时使用 `rel="noopener noreferrer"`。
- 上传图片必须校验扩展名、MIME、文件签名、尺寸和大小，并使用随机文件名。
- 禁止 SVG 上传或必须进行严格净化，首期建议只允许 JPG、PNG、WebP。
- 所有编辑接口按当前登录用户确定 `userId`，不能接受客户端传入任意目标用户 ID。
- OAuth subject、OpenID、Client Secret、Access Token 不得写入主页配置 JSON。
- 管理员隐藏主页必须保留原因、操作人和操作时间审计记录。
- 成员主页访问和编辑接口应增加合理的限流与缓存失效策略。

## 16. 状态与异常处理

| 场景 | 前端表现 |
| --- | --- |
| 成员列表加载中 | 展示与长条卡片同尺寸的骨架屏 |
| 没有匹配成员 | 显示“没有找到符合条件的成员”并提供清除筛选 |
| 主页未发布 | 本人显示编辑入口；其他成员显示“该主页暂未发布” |
| 主页被隐藏 | 显示统一不可访问页面，不暴露审核原因 |
| 背景图加载失败 | 使用系统默认渐变背景，不影响文字显示 |
| 模块数据请求失败 | 单模块显示重试，不阻塞整个主页 |
| 编辑保存冲突 | 提示页面已在其他位置更新，允许刷新或另存草稿 |
| 无成员权限 | 返回 403，并引导用户回到官网或个人工作台 |

## 17. 性能与可访问性

- 成员列表背景图使用响应式图片和懒加载，并声明固定宽高比避免布局跳动。
- 首屏主页横幅可预加载，其余模块按需加载。
- 列表分页建议每页 12～20 人。
- 主页渲染结果可按 `slug + version` 缓存，发布或下线时主动失效。
- 所有按钮触控区域不小于 44×44px。
- 卡片和模块支持键盘聚焦，并提供清晰焦点样式。
- 背景图必须叠加遮罩，确保文字对比度不低于 4.5:1。
- 图片提供有意义的 `alt`；纯装饰背景使用 CSS background，不重复朗读。
- 动画时长控制在 150～300ms，并遵守 `prefers-reduced-motion`。

## 19. 验收标准

### 19.1 成员广场

- 有效成员可以访问 `/members`，非授权用户无法读取列表数据。
- 每位有效成员都有一张可用卡片，未自定义用户显示默认卡片。
- 用户上传背景后，卡片正确裁切且文字始终可读。
- 搜索、筛选、分页、加载、空状态和失败状态完整。

### 19.2 成员主页

- 其他有效成员可以通过 `slug` 访问已发布主页。
- 主页只渲染启用且允许成员查看的组件。
- 最新博客只包含当前作者已发布文章。
- 页面不存在手机号、学号、OpenID、OAuth subject 等敏感字段。

### 19.3 主页编辑器

- 用户只能编辑自己的主页。
- 用户可以添加、隐藏、排序和配置预置组件。
- 预览和发布页面使用同一套渲染组件。
- 用户不能注入脚本、iframe、危险 HTML 或任意 CSS。

### 19.4 私密设置

- OAuth、QQ 绑定及密码修改从公开主页完全移除。
- `/settings/account` 只能由本人访问。
- 公开成员接口和浏览器网络响应中均不出现绑定标识原值。

### 19.5 响应式与主题

- PC、平板和手机均无水平滚动或内容遮挡。
- 深浅色模式下文字、边框和交互状态清晰可见。
- 所有关键交互支持键盘操作，触控区域符合 44px 最小尺寸。

## 20. 暂不纳入首期

- 面向互联网访客完全公开的个人主页。
- 用户上传任意 HTML、JavaScript、iframe 或 CSS。
- 私信、关注、好友关系和社交动态流。
- 自定义域名。
- 第三方博客平台自动同步。
- 未经审核的第三方组件市场。

## 21. 需要评审确认的问题

1. “有效社团成员”具体包含哪些成员状态。
2. 成员主页是否始终仅限社团成员访问，后续是否允许用户选择互联网公开。
3. 未发布主页的成员是否继续出现在广场；本文档建议显示默认卡片。
4. 部门、职位、奖项和活动数据是否允许用户分别控制公开。
5. 主页发布是否需要管理员审核；本文档首期建议先发布后治理，违规时隐藏。
6. 主页 `slug` 是否允许用户修改，以及修改后的旧地址是否需要重定向。
