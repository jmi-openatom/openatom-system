# 社团管理系统 PRD

## 1. 文档信息

- 产品名称：社团管理系统
- 文档版本：V1.1
- 输出日期：2026-05-07
- 适用范围：高校/组织内社团全生命周期管理

## 2. 产品背景

传统社团管理常依赖微信群、Excel、纸质登记与人工审批，存在如下问题：

- 社员入会、退会流程分散，记录不统一
- 面试安排依赖人工通知，过程不可追踪
- 审核节点多，责任人不清晰
- 社团干部权限混乱，容易越权操作
- 社员档案、活动参与、任职记录缺少统一沉淀

需要建设一套覆盖社团招新、入会、在会、转岗、退会、权限管理与后台运营的系统，支持标准化流程与数据沉淀。

## 3. 产品目标

- 建立统一的社团成员生命周期管理平台
- 实现入会申请、审核、面试、录用、入社、退社的线上闭环
- 通过 RBAC 模型实现精细化权限控制
- 提供用户管理、社团管理、岗位管理、审批管理、面试管理等核心能力
- 为后续活动管理、考核管理、通知管理、数据分析预留扩展空间

## 6. 核心业务流程

### 6.1 入会流程

1. 用户注册并实名认证/学号认证
2. 用户浏览社团招新计划
3. 用户填写入会申请表
4. 系统校验是否允许重复申请、是否满足基础条件
5. 进入初审
6. 初审通过后进入面试安排
7. 面试官打分并提交评语
8. 终审给出录用、备选、淘汰结果
9. 录用后生成成员记录并分配默认角色
10. 发送站内信/短信/邮件通知

### 6.2 退社流程

1. 社员发起退社申请
2. 填写退社原因与交接说明
3. 部门负责人审核
4. 社团管理员确认
5. 系统执行退社，回收岗位与权限
6. 保留成员历史档案，不做物理删除

### 6.3 转岗/升职流程

1. 管理员发起岗位调整
2. 选择成员、目标部门、目标岗位
3. 审核通过后生效
4. 更新成员角色、权限与任职记录

## 8. 非功能需求

### 8.1 安全性

- 密码加密存储
- 接口鉴权
- 防重放、防刷接口
- 权限校验到接口级
- 敏感字段脱敏展示
- 系统版本追溯：前端页脚自动展示当前构建版本/提交哈希
- 审计日志保留

### 8.2 性能

- 列表接口支持分页
- 支持按社团、状态、时间维度查询
- 高频接口响应目标小于 300ms

### 8.3 可用性

- 关键流程有状态提示
- 审核流程支持异常恢复
- 失败操作有错误码与提示语

### 8.4 可扩展性

- 审核流可配置
- 表单字段可扩展
- 角色权限支持动态配置

## 10. 状态机建议

### 10.1 申请状态机

- 草稿 -> 已提交
- 已提交 -> 初审通过 / 初审驳回
- 初审通过 -> 已安排面试
- 已安排面试 -> 已面试
- 已面试 -> 终审通过 / 候补 / 淘汰
- 任意可撤回节点 -> 已取消

### 10.2 成员状态机

- 待入社 -> 试用中 -> 正式成员
- 正式成员 -> 暂停资格 / 已退社 / 已毕业
