

# AstrBot + NapCat QQ群机器人 Web后台管理系统需求文档

## 1. 项目背景

本项目基于 **AstrBot + NapCat + OneBot v11** 构建 QQ 群机器人系统。AstrBot 支持多平台接入、插件扩展和可视化管理面板；NapCat 是基于 NTQQ 的 OneBot 实现，可作为 QQ 侧消息与群管理能力的接入层。([AstrBot](https://docs.astrbot.app/?utm_source=chatgpt.com "AstrBot"))

系统需要在保留现有机器人全部功能的基础上，新增 Web 后台，实现对机器人加入的 QQ 群进行统一管理、功能开关、成员管理、群公告、入群申请、欢迎语等能力。

## 2. 项目目标

建设一个 QQ 群机器人后台管理系统，实现：

1. 保留 AstrBot 现有所有聊天、插件、AI 对话等能力。

2. 接入 NapCat，实现 QQ 群消息收发与群管理能力。

3. 开发 Web 后台，可查看机器人加入的所有群聊。

4. 支持按群配置机器人功能是否启用。

5. 支持群成员、群主、管理员信息展示。

6. 支持群管理操作，包括禁言、全体禁言、公告、入群申请、新人欢迎等。

7. 支持权限控制，避免普通后台用户误操作群管理功能。

## 3. 系统架构

系统由以下模块组成：

```text
QQ / QQ群
   ↓
NapCat
   ↓ OneBot v11 / WebSocket / HTTP API
AstrBot
   ↓
自定义群管插件 / 后端服务
   ↓
Web 管理后台
```

### 3.1 AstrBot

负责机器人核心能力，包括：

- AI 对话

- 插件系统

- 群聊消息处理

- 指令处理

- 机器人基础配置

- 与 OneBot v11 适配器通信

AstrBot 官方文档说明其支持 QQ、企业微信、飞书、Telegram 等平台，并具备插件扩展和 WebUI 管理能力。([AstrBot](https://docs.astrbot.app/?utm_source=chatgpt.com "AstrBot"))

### 3.2 NapCat

负责 QQ 协议侧接入，提供 OneBot v11 能力，包括：

- QQ 登录

- 群消息接收

- 私聊消息接收

- 群成员信息获取

- 群管理 API 调用

- 文件上传相关能力

NapCat 支持 OneBot v11，并提供文件处理能力；其新版本还提供 Stream API 用于大文件上传、下载场景。([LangBot Documentation](https://docs.langbot.app/zh/usage/platforms/qq/aiocqhttp/napcat?utm_source=chatgpt.com "NapCat"))

### 3.3 Web 后台

负责给管理员提供可视化管理界面，包括：

- 群列表

- 群成员管理

- 群权限管理

- 群公告管理

- 入群申请处理

- 新人欢迎语配置

- 每个群的机器人启用状态配置

## 4. 用户角色

### 4.1 超级管理员

拥有系统最高权限，可进行：

- 管理所有群聊

- 管理后台账号

- 配置机器人全局参数

- 管理所有插件

- 查看操作日志

- 修改系统配置

### 4.2 群管理员

只能管理被授权的群，可进行：

- 查看指定群信息

- 查看群成员

- 禁言群成员

- 全体禁言

- 发布群公告

- 处理入群申请

- 配置欢迎语

- 开关机器人功能

### 4.3 只读用户

只能查看数据，不能执行群管理操作。

## 5. 功能需求

## 5.1 保留现有机器人功能

系统需完整保留当前 AstrBot 已有功能，包括但不限于：

- AI 聊天

- 群聊回复

- 私聊回复

- 插件功能

- 指令功能

- 知识库能力

- 上下文管理

- 机器人人格配置

- 关键词触发

- 现有配置项

要求新增 Web 后台后，不影响原有功能运行。

## 5.2 群聊管理

### 5.2.1 群列表

后台应展示机器人已加入的所有 QQ 群。

字段包括：

| 字段     | 说明          |
| ------ | ----------- |
| 群号     | QQ 群唯一 ID   |
| 群名称    | QQ 群名称      |
| 群人数    | 当前群成员数量     |
| 群主     | 群主 QQ / 昵称  |
| 管理员数量  | 群管理员数量      |
| 机器人状态  | 启用 / 禁用     |
| 最后活跃时间 | 最近一次收到群消息时间 |

### 5.2.2 群详情

点击群聊后进入详情页，展示：

- 群基本信息

- 群主信息

- 管理员列表

- 群成员列表

- 群公告列表

- 机器人配置

- 最近操作日志

## 5.3 群成员管理

### 5.3.1 成员列表

展示当前群所有成员。

字段包括：

| 字段     | 说明              |
| ------ | --------------- |
| QQ号    | 成员唯一 ID         |
| 昵称     | QQ 昵称           |
| 群名片    | 群内备注            |
| 身份     | 群主 / 管理员 / 普通成员 |
| 入群时间   | 加入群时间           |
| 最后发言时间 | 最近一次发言          |
| 禁言状态   | 正常 / 禁言中        |

### 5.3.2 成员操作

支持以下操作：

- 禁言成员

- 解除禁言

- 设置禁言时长

- 踢出群聊，二期可做

- 查看成员历史发言，二期可做

禁言时长支持：

- 1 分钟

- 10 分钟

- 1 小时

- 1 天

- 自定义秒数 / 分钟数

## 5.4 全体禁言

后台支持对指定群开启或关闭全体禁言。

操作要求：

- 开启全体禁言前需要二次确认

- 关闭全体禁言需要记录操作人

- 只有超级管理员或授权群管理员可操作

- 操作结果需要写入日志

## 5.5 群公告管理

### 5.5.1 发布公告

后台支持向指定 QQ 群发布群公告。

公告内容包括：

- 公告标题

- 公告正文

- 附件，可选

- 图片，可选

- 发布时间

- 发布人

NapCat 提供文件处理能力，大文件传输场景建议使用其 Stream API。([NapCatQQ](https://napneko.github.io/develop/file?utm_source=chatgpt.com "NapCat 文件处理框架指南"))

### 5.5.2 公告列表

后台展示已发布公告：

| 字段    | 说明         |
| ----- | ---------- |
| 公告 ID | 系统内部 ID    |
| 群号    | 所属群        |
| 标题    | 公告标题       |
| 内容摘要  | 公告前 50 字   |
| 附件    | 是否包含附件     |
| 发布人   | 后台操作人      |
| 发布时间  | 发布时间       |
| 状态    | 已发布 / 发布失败 |

### 5.5.3 公告管理

支持：

- 查看公告详情

- 删除公告

- 重新发布失败公告

- 查看附件

- 查看发布结果

## 5.6 入群申请管理

后台支持查看和处理入群申请。

功能包括：

- 查看申请人 QQ

- 查看申请人昵称

- 查看申请理由

- 查看申请时间

- 同意入群

- 拒绝入群

- 拒绝并填写理由

- 自动通过规则配置，二期可做

## 5.7 新人欢迎消息

当有新成员加入群聊时，系统可以自动发送欢迎语。

### 5.7.1 欢迎语配置

每个群可单独配置：

- 是否启用新人欢迎

- 欢迎语内容

- 是否 @ 新成员

- 是否发送图片

- 是否发送附件

- 延迟发送时间

示例：

```text
欢迎 {nickname} 加入本群！
请先阅读群公告，遵守群规。
```

支持变量：

| 变量             | 说明     |
| -------------- | ------ |
| `{nickname}`   | 新成员昵称  |
| `{qq}`         | 新成员 QQ |
| `{group_name}` | 群名称    |
| `{group_id}`   | 群号     |

## 5.8 按群启用机器人

后台支持对每个群单独配置机器人状态。

状态包括：

| 状态     | 说明          |
| ------ | ----------- |
| 启用     | 机器人正常响应     |
| 禁用     | 机器人不回复该群消息  |
| 仅管理员可用 | 只有群管理员触发时响应 |
| 仅指令模式  | 只有使用指令时响应   |
| 静默监听   | 记录消息但不回复    |

## 5.9 插件功能开关

每个群可以单独设置插件是否启用。

例如：

| 插件    | 群A  | 群B  |
| ----- | --- | --- |
| AI聊天  | 启用  | 禁用  |
| 关键词回复 | 启用  | 启用  |
| 入群欢迎  | 启用  | 禁用  |
| 群管功能  | 启用  | 启用  |
| 娱乐功能  | 禁用  | 启用  |

## 5.10 后台首页

首页展示系统概览：

- 当前机器人在线状态

- NapCat 连接状态

- AstrBot 运行状态

- 已加入群数量

- 今日消息数

- 今日群管理操作数

- 待处理入群申请数量

- 最近异常日志

## 5.11 操作日志

所有后台操作必须记录日志。

日志字段：

| 字段   | 说明                  |
| ---- | ------------------- |
| 操作人  | 后台账号                |
| 操作类型 | 禁言 / 公告 / 审核 / 配置修改 |
| 操作对象 | 群 / 成员 / 公告         |
| 操作内容 | 具体操作                |
| 操作时间 | 时间                  |
| 操作结果 | 成功 / 失败             |
| 失败原因 | 如有                  |

## 6. 权限要求

### 6.1 后台权限

权限分为：

- 超级管理员

- 群管理员

- 只读用户

### 6.2 群操作权限

执行群管理操作前，系统需要判断：

1. 后台用户是否有该群权限。

2. 机器人是否是该群管理员。

3. 目标成员是否为群主或管理员。

4. 当前操作是否被系统允许。

如果机器人不是群管理员，则禁言、全体禁言、公告等操作可能失败，需要在后台提示。

## 8. 接口需求

### 8.1 群列表接口

```http
GET /api/groups
```

返回：

```json
{
  "code": 0,
  "data": [
    {
      "group_id": "123456",
      "group_name": "测试群",
      "member_count": 200,
      "owner_id": "10001",
      "bot_enabled": true
    }
  ]
}
```

### 8.2 群成员接口

```http
GET /api/groups/{group_id}/members
```

### 8.3 禁言成员接口

```http
POST /api/groups/{group_id}/members/{user_id}/mute
```

参数：

```json
{
  "duration": 600
}
```

### 8.4 全体禁言接口

```http
POST /api/groups/{group_id}/mute-all
```

参数：

```json
{
  "enabled": true
}
```

### 8.5 发布公告接口

```http
POST /api/groups/{group_id}/announcements
```

参数：

```json
{
  "title": "公告标题",
  "content": "公告内容",
  "attachments": []
}
```

### 8.6 入群申请处理接口

```http
POST /api/groups/{group_id}/requests/{request_id}/handle
```

参数：

```json
{
  "approve": true,
  "reason": ""
}
```

### 8.7 群功能开关接口

```http
PUT /api/groups/{group_id}/config
```

参数：

```json
{
  "bot_enabled": true,
  "mode": "command_only",
  "welcome_enabled": true
}
```

## 10. 非功能需求

### 10.1 安全性

- 后台必须登录后访问

- 所有接口需要鉴权

- 高危操作需要二次确认

- 禁言、全体禁言、踢人等操作必须记录日志

- 不允许普通用户越权管理未授权群

### 10.2 稳定性

- NapCat 断线后自动重连

- AstrBot 异常时后台显示状态

- 群管理 API 调用失败时需要返回明确错误

- 定时同步群列表和群成员

### 10.3 可扩展性

系统需支持后续扩展：

- 多机器人账号

- 多平台接入

- 更多群管能力

- 更多插件配置

- 群消息审计

- 敏感词管理

- 自动审核入群申请

### 10.4 性能要求

- 群列表加载时间不超过 2 秒

- 成员列表支持分页

- 日志支持分页查询

- 附件上传需限制大小

- 大群成员同步需异步处理

## 11. 验收标准

### 11.1 基础验收

- AstrBot 原有功能正常使用

- NapCat 能正常连接 QQ

- 机器人能正常接收群消息

- 机器人能正常发送群消息

- 后台能显示机器人加入的群

### 11.2 群管理验收

- 能查看群成员

- 能识别群主和管理员

- 能禁言普通成员

- 能解除禁言

- 能开启 / 关闭全体禁言

- 能发布群公告

- 能携带附件发布公告

- 能处理入群申请

- 新成员加入后能发送欢迎语

### 11.3 配置验收

- 能按群启用或禁用机器人

- 能按群配置欢迎语

- 能按群配置插件开关

- 配置修改后立即生效或在明确提示后生效

- 普通后台用户不能管理未授权群

- 只读用户不能执行禁言、公告、审核等操作

- 所有操作均有日志记录

- 机器人权限不足时，后台能明确提示失败原因

## 13. 风险说明

1. QQ 群管理能力依赖 NapCat 和 OneBot 接口支持，部分能力可能因 QQ 协议变化失效。

2. 机器人必须是群管理员，才能执行禁言、全体禁言、公告等管理操作。

3. 公告附件能力需根据 NapCat 当前文件上传接口实际支持情况确认。

4. 大群成员同步可能较慢，需要异步任务或缓存。

5. 高危操作必须有权限控制和操作日志，避免误操作。
